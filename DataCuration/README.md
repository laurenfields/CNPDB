# cNPDB Data Curation

Everything for QC-ing the database and keeping it current with the literature.

> **The database itself is never written by these tools.** `Assets/20260418_cNPDB.xlsx`
> only changes when a curator deliberately merges reviewed additions.

## Layout — four kinds of thing

| Folder | What it is | Who writes it |
|--------|-----------|---------------|
| `*.py` (here) | **The tools.** Five commands, below. | humans |
| `reports/` | **Human-readable findings.** Read these first. | humans (from tool output) |
| `outputs/` | **Machine-generated data.** Regenerated by CI — *do not hand-edit.* | CI |
| `staging/` | **Proposed DB additions awaiting curator sign-off.** | humans + tools |
| `NeuroPepDatabases/` | Source data (NeuroPep export used in the original build). | frozen |

## The outputs, and which are automatic

| File in `outputs/` | What it holds | Refreshed by |
|--------------------|---------------|--------------|
| `qc_flagged_issues.csv` | Every QC issue, with a `severity` column (33 high / 100 medium / 117 low) | **CI** — `curation-refresh` on any DB or tool change |
| `audit_overlooked_neuropep.csv` | 59 sequences missing from cNPDB entirely | **CI** — same |
| `audit_os_underrepresented.csv` | 216 species gaps vs NeuroPep | **CI** — same |
| `new_papers_<date>.csv` | Raw scan: papers not yet seen or cited (the haystack) | **CI** — `literature-mining`, monthly |
| `shortlist_<date>.csv` | Crustacea-only, discovery-ranked papers to screen (the needles) | **CI** — same |
| `os_tissue_gaps.csv` | 134 peptides where the literature reports a species/tissue the DB omits | **CI** — same |
| `seen_dois.txt` | Papers already surfaced, so each month shows only what's new | **CI** — same |

Everything above regenerates itself. The only thing a human must produce is the
**full-text screen** of the shortlist — reading papers and their SI cannot be automated.

### Why staging exists (the PTM problem)
Most crustacean neuropeptides are C-terminally amidated, which shifts the monoisotopic
mass by −0.984 Da. The NeuroPep export doesn't record it, and **guessing would recreate the
exact class of mass error the QC pass found**. So staged rows carry the *unmodified* mass,
an `_mass_if_amidated` alternative, a `_likely_amidated` family hint, and a `_VERIFY` column.
Once a curator fills `PTM` / `Existence` / `Tissue`:

```bash
python -m DataCuration.backfill finalize     # recomputes mass from the confirmed PTM
```

## The five tools

```bash
python -m DataCuration.cnpdb_qc --out                 # QC the database -> outputs/
python -m DataCuration.coverage_audit                 # cross-check vs NeuroPep -> outputs/
python -m DataCuration.lit_mining discover --since 2025-01-01 \
    --seen DataCuration/outputs/seen_dois.txt --update-seen \
    --out DataCuration/outputs/new_papers_$(date +%F).csv \
    --shortlist DataCuration/outputs/shortlist_$(date +%F).csv
python -m DataCuration.os_tissue_completeness         # species/tissue gaps -> outputs/
python -m DataCuration.backfill                       # stage additions -> staging/
```

`lit_mining references --limit 50` back-fills citations for existing peptides.
Set `NCBI_API_KEY` / `NCBI_EMAIL` to raise the PubMed rate limit; Europe PMC needs no key.

## Scope: Crustacea only
`lit_mining.classify_relevance()` keeps a paper only when it is **both** crustacean **and**
neuropeptide-related — this is what cuts a ~316-paper scan to a ~12-paper shortlist. A
non-crustacean model organism (Drosophila, mouse) only vetoes a paper when *no* crustacean
signal is present, so crustacean-vs-insect comparisons survive.

## Reading the gap lists honestly
`os_tissue_gaps.csv` reports **screening candidates, not facts**: a species named in a paper
that mentions a peptide is not proof the peptide occurs there. Every row cites its supporting
paper. Peptides under 6 aa are skipped (short string matches retrieve too much noise). For the
8 species NeuroPep covers, `audit_os_underrepresented.csv` is the higher-confidence source.

## Adding entries — checklist
1. **Bare residues** in `Sequence`; every modification goes in `PTM` (never inline like `(d)`).
2. Compute `Monoisotopic Mass` with `utils.peptide_properties.monoisotopic_mass` — never by hand.
3. Fill `OS`, `Family` (controlled list), `Existence`, `DOI`.
4. `python -m pytest` and `python -m DataCuration.cnpdb_qc` must both pass.
5. Regenerate the `.parquet` and FASTA so all three artifacts stay in sync.
